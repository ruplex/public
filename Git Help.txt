Стандартные команды:
git clone "ссылка на репозиторий" - клонирует репозиторий;

git remote add "имя" "путь к репозиторию" - добавить репозиторий;

git init - иницилизирует файлы конфиги;

git add "имя файла";
git add . ;
git add --all;

git commit -m 'коммит'

git push (-f) "имя репозитория" master;

git pull - скачивает измененные файлы

git rm - удаляет файлы ;

Дополнительно:
git config core.autocrlf false - решает проблемы с CRLF;

git stash - обновляет коммиты;
git reset --hard HEAD - обновляет коммиты;

git help - помощь;

Если команда git pull не работает:
git add --all
git commit -m ''
git pull

Работа с ветками:

	Добавление:
	git checkout -b имя_ветки
	git push origin имя_ветки

	Слияние:
	git merge origin/имя_ветки
	git pull origin имя_ветки
	
	Удаление:
	git branch -d имя_ветки
	git push origin --delete имя_ветки
	
	Откат
	git reset --hard ид_ветки
	git reset --hard origin/имя_ветки
	git push -f origin имя_ветки

Работа с метками:
	
	git tag - просмотр существующих меток;
	
	Создание аннотированной (содержит информацию (имя, описание) отправителя) метки:
	git tag -a имя_метки -m 'описание' - -a создает аннотированую метку, -m меточное описание;
	
	Создание легковесных меток:
	git tag "имя_метки" - для легковетных меток не использовать параметры -a, -s или -m:
	
	Подписанные (аннотированная метка содержит подпись GPG(электронная цифровая подпись), если есть ключ) метки:
	git tag -s имя_метки -m 'описание' - -s подписывает метку, -m описание;
	
	git show "имя_метки" - просмотреть прикрепленную электронную цифровую подпись (GPG);
	
	Верификация(проверка):
	git tag -v "имя_метки" - эта команда использует электронную цифровую подпись(GPG) для проверки подписи;
